Block & Inline / Classes & Id

Use div as container for other HTML elements like p.

Can't use Block-Level-Elements inside of Inline-Elements.

The P-Element defines a Paragraph and therefore is a Block-Level-Element.

The Span-Element is an Inline-Container and can be used inside of P-Elementsto mark up a part.

An Element can have more than one class, then it's just a space between like: class="style1 style2"

Id on Element can't start with a number nor have whitespaces. It's unique and can be used for bookmarks like here: To Bookmark (href="#bookmark")

Layout & Responsive

The Semantic Elements used to create a good HTML Layout:

  • Header Element defines the header for the whole document or just a section
  • Nav Element defines a list of navigational links
  • Section Element defines a whole section in a document
  • Article Element defines an independent content
  • Aside Element defines content at the side of other content (sidebar)
  • Footer Element defines the bottom end of a document
  • Details Element defines additional content that can a user can open and close on demand
  • Summary Element defines the heading for the Details Element

The Techniques for a good HTML Layout:

  1. CSS Framework
  2. CSS Float
  3. CSS Flexbox
  4. CSS Grid

Semantic Elements

Non Semantic Elements tell nothing about its content (i.e. div, span)

Semantic Elements on the other side clearly defines what the content is (i.e. form, table, article, header, footer)

  1. Section Element:
    • Thematic Grouping of Content
    • Usually starts with Heading
    • Can be used for: Chapters / Introduction / News Items / Contact Information
    • Web Page is normally split into Sections for Introduction, Content & Contact Information
    • There can be Sections with Articles in them
  2. Article Element:
    • Specifies indepedent and self-contained Content
    • Should make sense on its own (can be distributed without the rest of the website)
    • Can be used for: Forum posts / Blog posts / User comments / Product Cards / Newspaper Articles
    • There can be Articles with Sections in them
  3. Header Element
    • Represents a container for Introductory Content or Set of Navigational Links
    • Typically contains: One or more Heading Elements (h1 - h6) / Logo or Icon / Authorship Information
    • The Website can have multiple Header Elements
    • Can't have Header Elements inside Header / Footer / Address Elements
  4. Footer Element
    • Defines the Footer for a Document or Section
    • Typically contains: Authorship- / Copyright- / Contact- Information / Sitemap / Back to Top Link / Related Websites
    • The Website can have more than just one Footer
  5. NAV Element
    • Defines a Set of Navigational Links
    • Should only contain the Major Block of Navigational Links
  6. Aside Element
    7. This should float besides the Content
    • Defines Content besides the Content it is placed (i.e. Sidebar)
    • Should be directly related to the surrounding Content
  7. Figure & Figcaption Element
    • Figure Element specifies self-contained Content like Illustrations, Diagrams, Photos, Code Listings etc.
    • Figcaption Element defines the Caption for Figure Element -> Can be placed as first or last Child
    • IMG Element specifies the actual Image / Illustration
  8. Main Element
    • Specifies the Main Content of a Document
    • The Content inside should be unique and not contain any repeated Content across the Page
    • No Sidebars, Navigational Links, Copyright Information, Site Logos, Search Forms
    • There must be only one Main Element and can not be a Descendant of Article, Aside, Footer, Header & Nav
  9. Time Element
    • Defines a specific Time or Datetime
    • Can have the datetime="2022-02-22 22:22" Attribute
    • Browsers can offer Date reminders through the User's Calendar and Search Engines produce smarter Results
